# Code Review Skill

Perform comprehensive code reviews with structured feedback.

Source: claudedirectory.org/skills/review

## Usage
/review [file or PR]

## Behavior
1. Analyze code changes
2. Check for:
   - Logic errors and bugs
   - Security vulnerabilities
   - Performance issues
   - Code style violations
   - Missing tests
3. Provide actionable feedback

## Review Categories

### Critical (must fix)
- Security vulnerabilities, data loss risks, breaking changes

### Important (should fix)
- Performance problems, logic errors, missing error handling

### Suggestions
- Code style, better naming, refactoring opportunities

## Output Format
## Code Review Summary
### Critical Issues
- [file:line] Description
### Important Issues
- [file:line] Description
### Suggestions
- [file:line] Description
### What's Good
- Positive feedback
